The AngularHost project is a micro frontend host application that loads multiple child applications. In this case, it loads the teams, staff, and role modules from the Angular18Client child applications.

Project Structure

The project is organized into several key folders:

Core

The core folder contains core components and services that are used throughout the application. These are typically singleton services and utility classes that provide essential functionality.

  • Core Components: These are reusable components that provide common UI elements or functionality.
  • Core Services: These services provide core functionality such as authentication, logging, and configuration management.

Models

The models folder contains TypeScript interfaces and classes that define the data models used in the application. These models represent the structure of the data that is exchanged between the frontend and backend services.

  • Data Models: Interfaces and classes that define the shape of the data used in the application.

Modules

The modules folder contains feature modules that encapsulate specific functionality of the application. Each module typically contains its own components, services, and routing configuration.

  • Teams Module: Manages the teams-related functionality.
  • Staff Module: Manages the staff-related functionality.
  • Role Module: Manages the role-related functionality.

Services

The services folder contains application-wide services that provide business logic and data access functionality. These services are typically injected into components and other services using Angular's dependency injection system.

  • Business Services: Services that contain business logic and interact with backend APIs.
  • Utility Services: Services that provide utility functions such as data formatting and validation.

Micro Frontend Architecture

The AngularHost project uses a micro frontend architecture to load and integrate multiple child applications. This approach allows for independent development and deployment of different parts of the application, improving scalability and maintainability.

In this project, the AngularHost application loads the following child applications:

  • Teams Module: Loaded from the Angular18Client child application.
  • Staff Module: Loaded from the Angular18Client child application.
  • Role Module: Loaded from the Angular18Client child application.

Each child application is developed and deployed independently, and the AngularHost application dynamically loads them as needed.
